The sound use case

In this use case your task is to be able to put code together in such a way that the harder you scream into the sound sensor, the more lights turn on.

Step 1: Connect a variable with the name "soundStateCurrent" to the sound sensor

Step 2: If the sound sensor reaches the threshold of 100 or higher, the first light on pin 13 should turn on. After 1 second it should automatically turn off again.

Step 3: If the sound sensor reaches the threshold of 400 or higher, the second light on pin 12 should turn on. After 1 second it should automatically turn off again.

Step 4: If the sound sensor reaches the threshold of 700 or higher, the second light on pin 11 should turn on. After 1 second it should automatically turn off again.

The button use case

In this use case your task is to be able to put code together in such a way that the longer you hold down the button, the more lights turn on.

Step 1: If the button on pin 7 is held down, the first led on pin 13 should turn on. It should reset after two seconds. Make sure that if the button isnt held down, the led does not turn on.

Step 2: If the button on pin 7 is still held down after two seconds, the second led on pin 12 should turn on. It should reset after two seconds. Make sure that if the button isnt held down, the led does not turn on.

Step 3: If the button on pin 7 is still held down after two seconds, the third led on pin 11 should turn on. It should reset after two seconds. Make sure that if the button isnt held down, the led does not turn on.
